Migration Notes for React Native

Navigation Structure

  • Use React Navigation with:
    • Stack Navigator (main navigation)
    • Drawer Navigator (side menu)
    • Tab Navigator (where applicable)
    • Deep linking configuration

Screen Mapping

  • Each Activity → React Native Screen Component
  • Fragments → Can be components or separate screens
  • Dialogs → Modal components

State Management

  • User authentication state
  • Navigation state
  • Current child/student context
  • Feature flags
  • Offline data caching

Key Libraries to Use

  • Navigation: @react-navigation/native, @react-navigation/stack, @react-navigation/drawer
  • State: Redux Toolkit / Zustand / Context API
  • API: Axios
  • Storage: @react-native-async-storage/async-storage or react-native-mmkv
  • Maps: react-native-maps
  • Images: react-native-fast-image
  • Video: react-native-video
  • Camera: react-native-image-picker, react-native-camera
  • Firebase: @react-native-firebase/app, @react-native-firebase/messaging, @react-native-firebase/analytics, @react-native-firebase/crashlytics
  • Payment: react-native-razorpay
  • UI: react-native-paper / react-native-elements / custom components

Platform-Specific Considerations

  • iOS: Native modules for specific features, App Store guidelines compliance
  • Android: Native modules, Google Play guidelines compliance
  • Permissions: react-native-permissions for handling platform-specific permissions
  • Deep Linking: react-native-deep-linking for both platforms
